Student Database Management System (C)
A memory-efficient command-line application designed to manage academic records using dynamic Singly Linked Lists. This project focuses on low-level memory management and automated academic logic.

Key Features
Dynamic Record Management: Implements full CRUD (Create, Read, Update, Delete) operations using a linked list structure.

Flexible Academic Logic: Users define passing marks and grace thresholds at runtime to categorize students.

Automated Grading Engine:

Grace Marks: Automatically identifies students within a specific range of passing and applies grace marks.

Status Tracking: Dynamically updates student status as 'Pass' or 'Fail' based on subject performance.

Advanced Data Filtering: Generates specific reports for "Graced," "Failed," or "Passed" students with formatted table outputs.

Manual Memory Management: Includes a dedicated cleanup routine to deallocate the Heap memory and prevent leaks.

Technical Stack
Language: C

Data Structures: Singly Linked Lists, Structs, Pointers.

Concepts: Dynamic Memory Allocation (malloc/free), Function Scoping, and Pointer Manipulation.
